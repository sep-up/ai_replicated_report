# 数据统计卡片功能文档

## 1. 功能概述

本功能模块主要实现智能制造系统中的数据统计卡片，用于展示各类订单和工单的数量统计信息。通过直观的卡片形式，帮助用户快速了解生产、销售、采购和工单的状态分布情况。

## 2. 数据统计卡片结构

### 2.1 生产订单数卡片
- **创建**：生产订单的创建数量
- **生效**：生产订单的生效数量
- **完成**：生产订单的完成数量

### 2.2 销售订单数卡片
- **创建**：销售订单的创建数量
- **生效**：销售订单的生效数量
- **完成**：销售订单的完成数量

### 2.3 采购订单数卡片
- **创建**：采购订单的创建数量
- **生效**：采购订单的生效数量
- **完成**：采购订单的完成数量

### 2.4 生产工单数卡片
- **创建**：生产工单的创建数量
- **发放**：生产工单的发放数量
- **投入**：生产工单的投入数量
- **挂起**：生产工单的挂起数量
- **完成**：生产工单的完成数量

### 4.2 数据获取与处理

```javascript
import { 
  getProductionOrderStatistics, 
  getSalesOrderStatistics, 
  getPurchaseOrderStatistics, 
  getWorkorderStatistics 
} from '@/api/statistics'

export default {
  data() {
    return {
      productionOrders: {
        created: 0,
        effective: 0,
        completed: 0
      },
      salesOrders: {
        created: 0,
        effective: 0,
        completed: 0
      },
      purchaseOrders: {
        created: 0,
        effective: 0,
        completed: 0
      },
      workorders: {
        created: 0,
        released: 0,
        input: 0,
        suspended: 0,
        completed: 0
      },
      loading: true
    }
  },
  created() {
    this.fetchStatistics()
  },
  methods: {
    async fetchStatistics() {
      this.loading = true
      try {
        // 获取生产订单数据
        const productionRes = await getProductionOrderStatistics()
        if (productionRes.data.code === 200) {
          productionRes.data.data.forEach(item => {
            if (item.name === '创建') {
              this.productionOrders.created = item.quantity
            } else if (item.name === '生效') {
              this.productionOrders.effective = item.quantity
            } else if (item.name === '完成') {
              this.productionOrders.completed = item.quantity
            }
          })
        }

        // 获取销售订单数据
        const salesRes = await getSalesOrderStatistics()
        if (salesRes.data.code === 200) {
          salesRes.data.data.forEach(item => {
            if (item.name === '创建') {
              this.salesOrders.created = item.quantity
            } else if (item.name === '生效') {
              this.salesOrders.effective = item.quantity
            } else if (item.name === '完成') {
              this.salesOrders.completed = item.quantity
            }
          })
        }

        // 获取采购订单数据
        const purchaseRes = await getPurchaseOrderStatistics()
        if (purchaseRes.data.code === 200) {
          purchaseRes.data.data.forEach(item => {
            if (item.name === '创建') {
              this.purchaseOrders.created = item.quantity
            } else if (item.name === '生效') {
              this.purchaseOrders.effective = item.quantity
            } else if (item.name === '完成') {
              this.purchaseOrders.completed = item.quantity
            }
          })
        }

        // 获取生产工单数据
        const workorderRes = await getWorkorderStatistics()
        if (workorderRes.data.code === 200) {
          workorderRes.data.data.forEach(item => {
            if (item.name === '创建') {
              this.workorders.created = item.quantity
            } else if (item.name === '发放') {
              this.workorders.released = item.quantity
            } else if (item.name === '投入') {
              this.workorders.input = item.quantity
            } else if (item.name === '挂起') {
              this.workorders.suspended = item.quantity
            } else if (item.name === '完成') {
              this.workorders.completed = item.quantity
            }
          })
        }
      } catch (error) {
        console.error('获取统计数据失败:', error)
      } finally {
        this.loading = false
      }
    }
  }
}
```

## 5. 数据流程

1. **页面加载**：组件创建时调用 `fetchStatistics` 方法
2. **API调用**：通过 `getAllStatistics` 接口获取所有统计数据
3. **数据处理**：将返回的数据赋值给对应的数据模型
4. **页面渲染**：根据数据模型渲染统计卡片
5. **数据展示**：用户可以直观地查看各类订单和工单的数量统计

## 6. 技术实现要点

1. **响应式设计**：使用 CSS Grid 实现卡片的自适应布局
2. **模块化API**：将统计相关的API接口单独封装在 `statistics.js` 文件中
3. **数据模型**：为每种订单类型创建独立的数据模型，便于管理和展示
4. **错误处理**：在API调用中添加基本的错误处理机制
5. **性能优化**：使用整合接口减少API调用次数，提高页面加载速度



## 8. 总结

本功能模块通过清晰的卡片布局和完整的API接口，实现了智能制造系统中各类订单和工单的数量统计功能。用户可以通过直观的界面快速了解生产、销售、采购和工单的状态分布，为决策提供数据支持。同时，模块化的设计和良好的扩展性为后续功能的迭代和优化提供了便利。
